Events

Events are actions that are triggered either by mouse click, function calls, keyboard strokes, loading of images, submitting an HTML form, etc. Events are usually used in connection with functions. Functions will not work without a force that will trigger it.

Event Triggers

Events

Event Triggers

Events

onload/onunload: Triggered when entering/exiting a page; used to check browser info and handle cookies.

onfocus/onblur/onchange: Used for form field validation.

onsubmit: Validates all form fields before submission.

onmouseover/onmouseout: Used for animations, changing images when the mouse hovers over or leaves an element.

onclick: Used to link to another webpage or trigger an event, like opening a popup when a button is pressed.

⋆˙⟡ Popup Boxes ⋆.𐙚 ̊

Popup Boxes

Popup boxes are those windows that pop-out of your screen. They could be before a page is fully loaded, effect of an event on a page or before leaving a page. JavaScript has three kinds of popup boxes; alert box, user-input prompt box and confirm box. They're useful in warning the user, gathering user input or simply to verify something.

Alert Box

This is usually used when you want to make sure that the user reads your message such as warning or a greeting. Syntax: alert("message");

User-Input Prompt Box

Usually used when you want data to be prompted first before the user can enter a page. User will have to choose either “ok” or ”cancel” to continue. If the user chooses “ok”, the prompt box will return the encoded value, else, it'll return null. Syntax: prompt("message","defaultvalue");

Confirm Box

This is usually used when you want the user to verify or accept something. The user will have to choose either “ok” and “cancel”. If “ok” was chosen, the confirm box will return true; else, it will return false. Syntax: confirm("message");

Variable

- A variable is a temporary storage of values and expressions. Expressions are formulae that use mathematical symbols and numbers or a way to assign values to a variable. For example, x=1+3 is an expression and the value o=f x will hold the sum of one plus three which is four.

- Can detect user actions, such as mouse clicks, which HTML could not do alone.

- Can be combined with CSS to produce DHTML. DHTML can make site more flexible by adding effects such as message scrollers, mouse trails and falling snow.

In declaring a variable, there are certain rules to follow:

- Case-sensitive: X is different from x.

- Start: Must begin with a letter or underscore (_).

- Characters: Use letters, numbers, or underscores.

- Avoid: Don't use reserved JavaScript words.

- Automatic Declaration: Assigning a value to an undeclared variable automatically declares it.

- Variable Content: Can hold numbers, expressions, or string literals.

- String Literals: Text enclosed in quotation marks; numeric digits become strings.

- Unquoted Characters: JavaScript interprets unquoted characters as objects or functions.

- Declaration: Use var variableName = value;

- Loosely Typed: Data types don't need pre-definition; they adapt to the assigned value (integer, string, boolean, etc.). Can change during runtime.

- Boolean: A data type representing true or false values, used for logical decisions.

Arithmetic Operations

JS uses the same arithmetic operations as in most programming languages, like C. An arithmetic operator is a symbol used to tell the expression what kind of operation will be done.

Modulus is performed by the remainder operator (%), which calculates the remainder after one number (the dividend) is divided by another (the divisor)

Addition Symbol

The addition symbol (‘+’) can also be used to add strings or text values together – this is called concatenation. However, if you add a string and a numerical value, the result will still be a string.

Follow the order of precedence: innermost parentheses first, then outer parentheses, followed by multiplication, division, or modulus, and finally addition or subtraction.

The `++` and `--` operators can be used independently or in expressions, affecting the result; avoid using them unless you fully understand their impact.

Comparison Statements

Comparison statements are used to evaluate the quality or the difference between variables or values.

Logical Operators

Logical operators are used to determine the logic between expressions, variables, or values. It is usually used by two or more expressions to satisfy the needed conditions.

JS also has its own conditional operator that automatically assigns a value depending on the result of the condition.

Syntax for conditional statements: variablename=(condition)?value1:value2

Example using conditional statement: myFavorite=(color=="pink")?"Terrific":"Change color";

If `color == "pink"`, `myFavorite` becomes "Terrific"; otherwise, it is "Change color."

if: Checks if a condition is true.

if-else: Performs an action if true, otherwise another action.

if-else if.else: Checks multiple conditions for execution.

switch: Selects one of many code blocks to run.

The `if` block executes the code within braces if the specified condition is true. If the code to be executed consists of only a single line, you can omit the braces for simplicity, but for multiple statements, braces are necessary to group the code together.

else if: Executes code if prior conditions are false.

else: Executes if no conditions are met (the last resort).

switch: Conditional statement like if, but considers a single variable. Use break to prevent fall-through.
